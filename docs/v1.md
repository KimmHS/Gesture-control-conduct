# Gesture Conduct v1 기능 및 개선점 정리

작성일: 2026-05-24

## 1. v1 목표

v1의 목표는 웹캠 기반 hand landmark tracking을 이용해 지휘 제스처를 음악 제어 명령으로 바꾸는 것이다. 현재 구현은 기존 WAV 재생 데모를 유지하면서, 양손 landmark mapping과 MIDI CC11 Expression 출력을 추가하는 방식으로 구성되어 있다.

핵심 데모는 **Two-hand Section Mixer**다.

```text
왼손 = 악기군 선택
오른손 = 선택된 악기군의 dynamics / cue 제어
MIDI = 선택된 악기군 channel에 CC11 Expression 전송
```

## 2. v1 구현 기능

### 2.1 양손 tracking 확장

기존 tracker는 가장 안정적인 active hand 하나만 앱 제어에 사용했다. v1에서는 기존 active hand 흐름을 유지하면서, `TrackingResult.hands`에 감지된 양손 정보를 같이 담는다.

구현 위치:

```text
src/conductor_demo/vision/tracker.py
```

주요 결과:

```python
tracking.hands["Left"]
tracking.hands["Right"]
```

새 mapping layer는 이 값을 사용하고, 기존 single-hand dynamics / WAV playback 흐름은 깨지지 않도록 유지했다.

### 2.2 왼손 section 선택

왼손 index fingertip landmark `8`이 화면 상단 zone 안에 들어오면 해당 악기군을 선택한다.

지원 section:

```text
STRINGS
WOODWINDS
BRASS
PERCUSSION
TUTTI
```

선택 안정화를 위해 dwell / hold 규칙을 둔다.

```text
left_dwell_seconds = 0.25
left_hold_seconds = 0.75
```

즉, 손가락이 zone에 잠깐 스쳐도 바로 바뀌지 않고, 왼손이 잠깐 사라져도 선택이 즉시 해제되지 않는다.

구현 위치:

```text
src/conductor_demo/game/two_hand_mapping.py
```

### 2.3 오른손 dynamics 제어

오른손 wrist motion span을 기준으로 intensity를 계산한다.

```text
작은 움직임 = SOFT
중간 움직임 = NORMAL
큰 움직임 = LOUD
```

내부 expression 값은 MIDI 범위에 맞춰 `30-127`로 변환한다.

```python
expression = int(30 + right_intensity * 97)
```

선택 section이 `TUTTI`면 전체 section expression을 같이 갱신하고, 특정 section이 선택되어 있으면 해당 section만 갱신한다.

### 2.4 오른손 downbeat cue

오른손 wrist가 짧은 시간 안에 강하게 아래로 움직이면 downbeat로 판단한다.

조건:

```text
수직 이동량이 충분히 큼
수직 이동량이 수평 이동량보다 큼
아래 방향 속도가 threshold 이상
cooldown 안에서는 중복 cue 방지
```

명령 예시:

```text
CMD_CUE_STRINGS
CMD_CUE_TUTTI
CMD_SET_STRINGS_EXPRESSION_112
```

### 2.5 visual overlay

기존 debug overlay 위에 section mixer overlay를 추가한다.

표시 내용:

```text
section zones
selected section highlight
section별 expression meter
left gesture / right gesture
current command
score / beat / target
```

구현 위치:

```text
src/conductor_demo/game/two_hand_mapping.py
src/conductor_demo/app/runner.py
```

### 2.6 MIDI 파일 inspect

`assets/audio/Symphony7_2.mid`의 track, channel, program, note count를 확인하는 CLI를 추가했다.

실행:

```bash
PYTHONPATH=src python -m conductor_demo.music.midi_inspect assets/audio/Symphony7_2.mid
```

확인된 MIDI 정보:

```text
type: 1
ticks_per_beat: 384
tracks: 13
```

현재 파일 기준 channel map:

```python
SECTION_CHANNELS = {
    "STRINGS": [7, 8, 10, 11, 12],
    "WOODWINDS": [0, 1, 2, 3, 4, 5],
    "BRASS": [],
    "PERCUSSION": [6],
    "TUTTI": [],
}
```

주의: 현재 MIDI 파일에는 brass로 분류할 channel이 명확히 없다. 그래서 `BRASS` zone은 visual/scoring에는 존재하지만 MIDI 출력은 no-op이다.

### 2.7 MIDI CC11 output

선택된 section의 expression 값을 MIDI CC11로 출력한다.

구현 위치:

```text
src/conductor_demo/music/midi_output.py
```

사용 예시:

```bash
python main.py --midi-out "IAC Driver Bus 1"
```

동작:

```text
STRINGS 선택 -> strings channel에 CC11 전송
WOODWINDS 선택 -> woodwinds channel에 CC11 전송
PERCUSSION 선택 -> percussion channel에 CC11 전송
TUTTI 선택 -> 모든 mapped channel에 CC11 전송
```

같은 값이 매 frame 반복 전송되지 않도록 `_last_values` cache로 중복 MIDI message를 줄였다.

### 2.8 scoring layer

오른손 downbeat가 발생했을 때만 점수를 계산한다.

구현 위치:

```text
src/conductor_demo/game/scoring.py
```

현재 script:

```python
SCRIPT = [
    ("STRINGS", "NORMAL"),
    ("STRINGS", "SOFT"),
    ("WOODWINDS", "SOFT"),
    ("BRASS", "LOUD"),
    ("STRINGS", "LOUD"),
    ("PERCUSSION", "LOUD"),
    ("TUTTI", "NORMAL"),
    ("TUTTI", "SOFT"),
]
```

점수 규칙:

```text
section + dynamics 모두 맞음 = 100
section만 맞음 = 75
dynamics만 맞음 = 55
둘 다 틀림 = 20
```

## 3. 실행 방법

### 3.1 macOS 권장 실행

MediaPipe webcam runtime은 Python 3.11 또는 3.12 권장이다.

```bash
cd /Users/kimjongha/Gesture-control-conduct
python3.12 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
python main.py
```

MIDI output:

```bash
python main.py --midi-out "IAC Driver Bus 1"
```

macOS에서는 **Audio MIDI Setup**에서 IAC Driver를 켜고 DAW나 software instrument로 연결해야 한다.

### 3.2 Windows 실행

Windows에서도 구조상 동작 가능하다.

```powershell
cd C:\path\to\Gesture-control-conduct
py -3.12 -m venv .venv
.\.venv\Scripts\activate
pip install -r requirements.txt
python main.py
```

카메라 index 변경:

```powershell
python main.py --camera-index 1
```

MIDI output은 macOS IAC Driver 대신 `loopMIDI` 같은 virtual MIDI port가 필요하다.

```powershell
python main.py --midi-out "loopMIDI Port"
```

### 3.3 카메라 없는 smoke test

```bash
python main.py --self-test --verbose
```

이 명령은 webcam을 열지 않고 MediaPipe tracker와 overlay pipeline이 시작되는지 확인한다.

## 5. v1 한계

### 5.1 MIDI는 소리 자체를 재생하지 않음

현재 구현은 MIDI file을 직접 synth로 렌더링하지 않는다. MIDI CC11 message를 외부 synth/DAW/virtual MIDI port로 보내는 구조다.

즉, 실제 악기 소리를 들으려면 다음 중 하나가 필요하다.

```text
DAW
software instrument
external MIDI synth
soundfont 기반 internal synth
pre-rendered section stems
```

### 5.2 `BRASS` channel 부재

현재 `Symphony7_2.mid`에서 brass로 볼 수 있는 channel이 명확하지 않다. 그래서 `BRASS`는 overlay와 scoring에는 남겨두었지만, MIDI output은 비어 있다.

개선하려면:

```text
brass track이 포함된 MIDI file 사용
현재 MIDI file의 특정 channel을 brass로 재분류
새 MIDI arrangement 제작
```

### 5.3 왼손 pointing은 ray가 아니라 cursor 방식

v1은 검지 방향 vector를 계산하지 않고, index fingertip 위치가 zone 안에 들어오는지 본다.

장점:

```text
구현이 안정적
설명이 쉬움
demo에서 즉시 이해 가능
```

단점:

```text
실제 지휘 동작보다는 UI 버튼 터치에 가까움
팔이 화면 상단으로 올라가야 함
```

### 5.4 downbeat rule은 heuristic

현재 downbeat는 wrist trajectory 기반 heuristic이다. 음악 beat grid와 완전히 동기화되어 있지는 않다.

### 5.5 audio backend는 여전히 WAV 중심

기존 `demo_song.wav` playback은 유지된다. MIDI 파일 기반 실제 orchestral playback은 v1 범위 밖이다.

## 6. 개선점 우선순위

### P0: demo 안정성

1. Camera permission / device guide 추가
   - macOS: System Settings > Privacy & Security > Camera
   - Windows: Settings > Privacy & security > Camera
   - `--camera-index` 탐색 helper 추가

2. 실행 전 environment check CLI 추가
   - Python version
   - MediaPipe import
   - OpenCV camera open
   - sounddevice output device
   - MIDI output ports

3. MIDI port listing 기능 추가
   - `python main.py --list-midi-ports`
   - 사용자가 정확한 `--midi-out` 이름을 알 수 있게 함

### P1: 제스처 품질

1. 왼손 pointing 개선
   - index MCP -> index tip vector로 pointing ray 계산
   - 화면 zone과 ray intersection 사용
   - fingertip cursor 방식과 ray 방식 toggle 제공

2. 오른손 dynamics calibration
   - 사용자별 comfortable motion span 반영
   - 현재 calibration 결과를 right-hand mapper에도 연결
   - hand scale 기반 reference span 조정 UI 제공

3. downbeat quality 개선
   - wrist velocity peak
   - direction reversal
   - acceleration
   - cooldown adaptive threshold

### P2: 음악 제어 강화

1. internal MIDI synth mode 추가
   - `fluidsynth` 또는 다른 soundfont 기반 player 검토
   - `.sf2` soundfont 경로 config 추가
   - MIDI file playback과 CC11 실시간 제어 통합

2. section stem playback 추가
   - strings.wav
   - woodwinds.wav
   - brass.wav
   - percussion.wav
   - section별 gain control

3. MIDI channel map config 파일화
   - 현재는 코드 상수
   - 추후 `config/midi_sections.json` 또는 YAML로 분리

### P3: 게임성

1. beat script를 MIDI timeline과 연결
   - 현재는 fixed script 반복
   - MIDI marker 또는 measure 기반 target 생성

2. 점수 feedback 개선
   - combo
   - timing accuracy
   - section accuracy
   - dynamics accuracy

3. practice mode 추가
   - section selection만 연습
   - dynamics만 연습
   - downbeat timing만 연습
